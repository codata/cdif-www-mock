The CDIF framework is composed of several core profiles that specify how to use existing standards to achieve interoperability in specific functional areas. These profiles are designed to be used together or independently, depending on the needs of the implementation.

### Key Profiles

*   **Discovery**: Based on DCAT and Schema.org, providing a common way to describe datasets and services across domains.
*   **Data Access and Licensing**: Using ODRL and Data Usage Vocabularies to standardize how access is requested and granted.
*   **Controlled Vocabularies**: Leveraging SKOS for representing and sharing the terminologies used to describe data.
*   **Data Integration**: Practical guidelines for structural integration of multi-domain data.
*   **Universals (Temporal/Spatial)**: Standard ways to represent common cross-domain concepts like time and location.
