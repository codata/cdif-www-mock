## What is CDIF?

The Cross Domain Interoperability Framework (CDIF) is a set of practical, implementation-level guidelines designed to improve data management practices within any community and lower the barriers to cross-domain data reuse. CDIF recommands standards and methodologies for achieving different levels of interoperability necessary for reusing data across diverse domains. It is built around several core profiles that address the essential functions for implementing cross-domain FAIR principles.

## Tools and Implementations

First published in May 2024 as an output from the WorldFAIR Project, CDIF is now developed and maintained by a group of invited experts, coordinated by CODATA. It is increasingly being implemented as part of projects in Europe, Africa, and globally, in a number of disparate domains. Increasingly, CDIF tools are being developed and made available. ## Core CDIF Profiles


CDIF provides a mechanism for publishing a core set of critical metadata.

A content model for structuring information, based on various existing metadata schemes.
A JSON-LD serialization for data exchange.
Workflows for publishing CDIF metadata, making resources easily discoverable and indexable by search engines using standard web technologies.

1. Discovery
CDIF supports basic resource discovery on the web using an implementation based on Schema.org and the DCAT RDF vocabulary.

2. Enabling Data Access and Use
CDIF defines machine-actionable methods for defining and evaluating access and usage conditions, using policies expressed with the Open Digital Rights Language (ODRL 2.2) standard.

3. Controlled Vocabularies
This profile outlines best practices for the management and publication of controlled vocabularies and other semantic artifacts, ensuring consistency of approach to enable greater interoperability across domains.

4. Data Integration
This profile provides resources that facilitate automated data integration through detailed descriptions of the variables and structures of a wide range of data. Uses the DDI-Cross Doamin Interoperability (DDI-CDI) specification for representing the conceptual, phhysical,and logical aspects of data.


5. Universals
This profile provides recommendations of ‘universal’ elements, including time, geography, and units of measurement, which are applicable across different domains for consistent interpretation and interoperability.

## Future Directions

Upcoming releases will look at supporting the detailed eescription of a broader range of data formats (NetCDF, HDF5, Parquet, JSON, XML, etc.), recommendations for describing provenance using the PROV Ontology, packaging of resources with RO Crates and other standards, description of data Context, and more. The next release is expected in mid-2026.
